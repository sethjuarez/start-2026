# Judson Althoff - concise demo description

The demo should tell one cohesive story, not four separate product demos.

Use a single company and a single business scenario as the red thread across the full narrative. A good example from the notes is Unilever and a claims management agent.

The flow should show:

1. **The employee experience** - start with how a person inside that company uses Copilot, Cowork, and Scout in their day-to-day work.
2. **How the agent was built** - then show the maker or developer perspective, building that same company-specific agent on the platform.
3. **How the agent improves a real business process** - show the agent operating inside an important workflow, with a clear business outcome such as the Unilever example that saved $2.4 million.
4. **How it is governed and optimized** - finish with Agent 365, showing management, security, observability, token usage, and FinOps for that exact same agent so the audience can see net ROI.

If helpful, add a light cyber moment in the middle to show security in context.

The key point is connectedness: one company, one scenario, one agent, shown across employee use, app creation, business impact, and operational governance. That is the Microsoft story Judson wants to tell.
